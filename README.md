# Сборка образов контейнеров (rootfs)

Репозиторий для сборки базовых rootfs-образов Linux-дистрибутивов, пригодных для контейнеров, chroot-окружений и дальнейшей упаковки в собственные образы. В первую очередь собирается для контейнеров LXC в Proxmox.

## Структура
Репозиторий разбит на директории по дистрибутивам. В каждой директории есть скрипт `make.sh`, предназначенный для создания образа диска. Так же в директоориях могут присудствовать вспомогательные файлы.

## Запуск
### Предварительная настройка
Зависимости:
1. bash
2. debootstrap

Если вы ранее не использовали user namespace для других программ, необходимо назначить вашему пользователю диапазоны UID и GID, которые будут использоваться в namespace. Эти команды разрешать использовать диапазон 1000000 - 1100000 для маппинга пользователей в namespace.
```sh
echo "$(whoami):1000000:100000" | sudo tee -a /etc/subuid
echo "$(whoami):1000000:100000" | sudo tee -a /etc/subgid
```

В некоторых дистрибутивах создание user namespace для не привилигированных пользователей отключен. `sysctl kernel.unprivileged_userns_clone` должен быть равен 1. Если нет, то
```sh
sudo sysctl -w kernel.unprivileged_userns_clone=0
```

### Сборка
Скрипты разрабатываются для работы без root-привилегий при помощи user namespace + mount namespace. Скорее всего они будут работать и под root, но данная возможность не тестируется. Перед запуском необходимого `make.sh` необходимо перейти в отдельный namespace:
```sh
unshare --user --fork --pid --mount --mount-proc --map-root-user --map-auto
```

По умолчанию сборка осуществляется в директории rootfs в папке со скриптом. Это можно переопределить, необходимо задать переменную `ROOTFS_DIR`.
```sh
ROOTFS_DIR=/tmp/debian_rootfs ./debian/make.sh
```

После завершения сборки выйдите из namespace нажав Ctrl+D или выполнив команду `exit`.
